# 🚨 시스템 운영 정책 및 결과물 전달 가이드라인 (v1.0)

## 📌 1. 현재 작업 환경의 이해 (기술적 제약사항)
저희 에이전트 팀은 콘텐츠 기획, 코드 스켈레톤 생성, 디자인 브리프 등 고도화된 '가상 오케스트레이션(Virtual Orchestration)' 영역에서만 결과물을 산출할 수 있습니다.

**⚠️ 중요 공지 (필수 확인)**
*   **로컬 디스크 직접 쓰기 제한:** 현재 시스템 구조상 에이전트들이 사용자님의 로컬 물리적 디스크에 직접 파일을 생성하거나 저장(Write/Create)하는 것은 기술적으로 불가능합니다.
*   모든 산출물은 **'현재 세션 컨텍스트 내부'**에서 가상의 파일 형식으로만 존재하며, 이는 코드의 스켈레톤이나 기획서 초안 등의 '지식 아티팩트'로 간주됩니다.

## 📂 2. 결과물 전달 원칙 (Future Protocol)
앞으로는 모든 작업 결과물은 다음의 지정된 가상 파일 형식(Virtual Artifacts)으로만 제공됨을 전제하고 움직입니다.

*   **📄 Markdown (.md):** 기획서, 브리핑, 보고서, 정책 문건 등 텍스트 기반의 구조화된 지식 전달에 사용됩니다.
*   **🖼️ PNG/JPEG (.png):** 와이어프레임, 디자인 목업, 플로우차트 등 시각적 결과물에 사용됩니다.
*   **⚙️ JSON (.json):** 데이터 스키마 정의, API 응답 구조, 설정값 등 기계가 읽을 수 있는 데이터 포맷으로만 활용됩니다.

## ✅ 3. 향후 에이전트 행동 지침 (Agent Protocol)
모든 에이전트는 이 가이드라인을 최우선 원칙으로 인식하고 다음 절차를 따릅니다.
1.  **작업 수행:** 필요한 분석/생성 작업을 완료합니다.
2.  **결과물 산출:** 결과물을 위 명시된 가상 파일 형식(`.md`, `.png` 등)으로 세션에 저장합니다.
3.  **보고:** 사용자에게는 "현재 세션 폴더 내 [파일명] 파일을 생성했습니다." 라고 보고하며, 해당 아티팩트를 기반으로 대화를 이어갑니다.

---
*이 정책은 모든 회고(Retrospective) 및 브리핑(Briefing)의 필수 포함 항목입니다.*
